# 코드 리뷰

## 활성화 조건

- "리뷰해줘", "코드 검토", "diff 봐줘" 요청
- PR 전 변경사항 점검

## 실행 절차

1. 리뷰 범위 파악 (`git diff`, 변경 파일 목록)
2. 보안 검사 (OWASP Top 10, 인젝션, 인증 우회)
3. 성능 분석 (N+1, 메모리 누수, 불필요 연산)
4. 접근성 검토 (해당 시, WCAG 2.1 AA)
5. 코드 품질 검토 (네이밍, 중복, 복잡도, 타입)
6. 심각도별 우선순위 보고서 생성

## 핵심 규칙

1. 발견 사항을 심각도 순으로 정렬
2. 모든 발견 사항에 `file:line`, 영향, 수정 방법 포함
3. 증거 없는 추측성 발견 사항 금지
4. 동작 변경 시 테스트 누락은 발견 사항
