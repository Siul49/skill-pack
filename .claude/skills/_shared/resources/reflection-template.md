# 회고 템플릿

## 파일 규칙

- 위치: `.claude/reflections/YYYY-MM-DD.md`
- 하루에 여러 작업이면 같은 파일에 append (구분선 `---`으로 분리)
- AI가 대신 작성하지 않는다 — 사용자의 답변을 기록한다

## 회고 질문 (매 작업 완료 후)

작업 완료 보고 직후 아래 질문을 던진다:

```
✍️ 회고 시간입니다. 아래 질문에 답해주세요:

1. 이 작업에서 뭘 배웠어?
2. 다음에 비슷한 작업을 한다면 다르게 할 게 있어?
3. 처음 예상과 달랐던 점이 있어?
```

### 간소화 (단순 작업)

단순 작업의 경우 질문을 1개로 줄일 수 있다:

```
✍️ 간단한 회고: 이 작업에서 하나라도 배운 게 있어?
```

### 회고 거부 대응

```
회고는 필수입니다. 간단하게라도 한 줄 답변해주세요.
→ 이 작업에서 뭘 배웠어?
```

## 기록 형식

```markdown
## HH:MM — {작업 요약 한 줄}

**배운 것**: {사용자 답변}
**다르게 할 것**: {사용자 답변}
**예상과 달랐던 점**: {사용자 답변}

---
```

### 단순 작업 기록 형식

```markdown
## HH:MM — {작업 요약 한 줄}

**배운 것**: {사용자 답변}

---
```

## 파일 헤더 (새 날짜 파일 생성 시)

```markdown
# 회고 — YYYY-MM-DD

---
```
